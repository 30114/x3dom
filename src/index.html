X3DOM-WebGL Examples

Below you'll find some examples for the X3DOM on WebGL implementation. For viewing them, you need a recent browser that supports WebGL.

Browser Support

Firefox

Firefox nightly builds support WebGL. You can download those here:
http://www.mozilla.org/developer/#builds

However, you need to turn on WebGL manually in the configuration. Type about:config in the address bar, then search for "webgl" and change the value of webgl.enabled_for_all_sites to true.

Webkit

Webkit nightly builds also support WebGL. You can download those here:
http://nightly.webkit.org/

Likewise, you still have to activate it. On Mac this works like follows:
defaults write com.apple.Safari WebKitWebGLEnabled -bool YES

Examples

  • Hello World: Minimal X3DOM example
  • Primitives: Box, Cone, Cylinder, Sphere
  • MultiView: Multiple X3D scenes in a single XHTML page
  • ImageFormat: Test different image format support
  • Texture: Model with multiple external textures
  • Tree Depth: Model with a mixed Transform/ Group hierarchy
  • Single Mesh: Model with a flat hierarchy
  • DEF/USE: Model with DEF/USE references
  • FanOut: Multiple animations with single TimeSensor and fanOut
  • Logo: An animated X3D Logo
  • MultiIndex: Model with multi-indexed geometry

Usage

Right now only a simple EXAMINE-mode is implemented. Use the left mouse button to rotate, the middle mouse button for pan and the right for zoom. WALK, FLY and LOOKAT will come later. If the web page has scroll bars and autoscrolling is enabled, pan currently does not work yet. As a workaround, you can disable autoscrolling by unchecking the Use autoscrolling checkbox in the browser options, as is shown in the screenshot below (for the Firefox case). Alternatively, use the Shift (rotate)/ Ctrl (Pan)/ Alt (Zoom) key in combination with a mouse button to switch between the three modes.
